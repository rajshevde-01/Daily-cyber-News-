# 🚨 Microsoft to stop Exchange 2016 / 2019 security updates in October
**Source**: BleepingComputer
**Link**: https://www.bleepingcomputer.com/news/microsoft/microsoft-exchange-2016-and-2019-esu-program-ends-in-october/
**Date**: 2026-07-22

## 📰 Summary of the Threat
The threat in question involves Microsoft's decision to discontinue security updates for Exchange 2016 and 2019 through the Extended Security Update (ESU) program in October. This decision targets organizations still relying on these versions of Exchange, leaving them vulnerable to potential security risks and exploits. The technical mechanism of this vulnerability is rooted in the lack of ongoing security patches and updates, which are essential for protecting against newly discovered vulnerabilities and threats.

Organizations using Exchange 2016 and 2019 will be exposed to an increased risk of cyberattacks, as attackers often exploit known vulnerabilities in outdated software. The absence of security updates means that any newly discovered vulnerabilities in these Exchange versions will not be patched, providing a potential entry point for attackers. This situation underscores the importance of maintaining up-to-date software and highlights the need for organizations to plan for the migration to supported versions of Exchange or alternative email solutions.

The impact of this decision will be most significant for organizations that have not yet migrated to newer, supported versions of Exchange or have not implemented alternative email solutions. These organizations must assess their current infrastructure and develop a plan to mitigate the risks associated with the discontinuation of security updates. This may involve upgrading to a supported version of Exchange, migrating to a cloud-based email solution, or implementing additional security controls to compensate for the lack of security updates.

## 🛠️ Actionable Defense & Mitigation Checklist
1. **Assess Current Exchange Infrastructure**: Conduct an immediate assessment of your current Exchange infrastructure to determine which versions are in use and their current patch levels. This will help identify potential vulnerabilities and inform migration or mitigation plans.
2. **Migrate to Supported Exchange Versions or Alternatives**: Develop a plan to migrate to a supported version of Exchange or an alternative email solution. This may involve upgrading to Exchange 2022 or migrating to Microsoft 365 or another cloud-based email service.
3. **Implement Additional Security Controls**: For organizations that cannot immediately migrate, implement additional security controls to mitigate the risks associated with the lack of security updates. This may include enhancing email filtering, implementing advanced threat protection solutions, and restricting access to Exchange servers.
4. **Regularly Audit and Monitor Exchange Servers**: Regularly audit and monitor Exchange servers for signs of compromise or suspicious activity. This may involve implementing logging and monitoring solutions, conducting regular security scans, and performing penetration testing to identify vulnerabilities.
5. **Develop an Incident Response Plan**: Develop an incident response plan that outlines procedures for responding to a potential security incident involving Exchange servers. This plan should include procedures for containment, eradication, recovery, and post-incident activities.
